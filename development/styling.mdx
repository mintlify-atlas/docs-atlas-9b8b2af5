ALESPI Solutions Web styles are driven by Tailwind CSS v3. You author styles in `input.css`, which passes through a PostCSS pipeline to produce a minified `output.css` that `index.html` loads. The project extends Tailwind with a custom brand color palette, a system font stack, and three named animations.

## Build pipeline

```
input.css  →  PostCSS (tailwindcss + autoprefixer)  →  output.css
```

The PostCSS configuration is minimal — it runs the Tailwind plugin followed by Autoprefixer:

```js
export default {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
}
```

`input.css` begins with the three Tailwind directives and then declares all custom component classes and animations:

```css
@tailwind base;
@tailwind components;
@tailwind utilities;

/* Custom component and animation definitions follow */
```

<Warning>
  Never edit `output.css` directly. It is generated by Tailwind and will be overwritten the next time you run a build command. Always make style changes in `input.css` and recompile.
</Warning>

## Compiling styles

Use the npm scripts defined in `package.json` to compile `output.css`.

<Tabs>
  <Tab title="One-time build">
    Produces a minified `output.css` suitable for deployment:

    ```bash
    npm run build
    ```

    This runs `tailwindcss -i ./input.css -o ./output.css --minify`.
  </Tab>
  <Tab title="Watch mode">
    Rebuilds `output.css` automatically whenever you save a change to `input.css` or any scanned file:

    ```bash
    npm run dev
    ```

    This runs `tailwindcss -i ./input.css -o ./output.css --watch`. Keep this running in a terminal while you develop.
  </Tab>
</Tabs>

## Content paths

Tailwind scans the files listed under `content` to determine which utility classes to include in `output.css`. Only classes that actually appear in these files are emitted.

```js
content: [
  "./index.html",
  "./src/**/*.{html,js}",
],
```

If you add a new partial under `src/`, its classes will be picked up automatically. If you move files outside these paths, you must add the new path to the `content` array.

## Custom font family

The `sans` family is overridden with a Windows/cross-platform system font stack. No external font files are loaded.

```js
fontFamily: {
  sans: ['Segoe UI', 'Tahoma', 'Geneva', 'Verdana', 'sans-serif'],
},
```

This stack ensures sharp rendering on Windows (Segoe UI), macOS/Linux fallbacks (Geneva, Verdana), and a generic sans-serif as the final fallback.

## Custom animations

Three named animations are registered in `tailwind.config.js`. Apply them with `animate-float`, `animate-slideUp`, and `animate-countUp` utility classes.

<AccordionGroup>
  <Accordion title="animate-float — continuous floating motion">
    Used on decorative background elements in the hero section. The element drifts 30 px downward and back over 6 seconds, repeating forever.

    ```js
    animation: {
      float: 'float 6s ease-in-out infinite',
    },
    keyframes: {
      float: {
        '0%, 100%': { transform: 'translateY(0px)' },
        '50%': { transform: 'translateY(30px)' },
      },
    },
    ```

    `input.css` also defines staggered variants directly: `.animate-float-1` (6 s), `.animate-float-2` (8 s, 1 s delay), and `.animate-float-3` (7 s, 2 s delay).
  </Accordion>
  <Accordion title="animate-slideUp — entrance from below">
    Used on hero text and section headings. Content slides up from 60 px below its final position while fading in, over 0.8 seconds. `input.css` defines this keyframe directly:

    ```css input.css
    @keyframes slideUp {
      from {
        opacity: 0;
        transform: translateY(60px);
      }
      to {
        opacity: 1;
        transform: translateY(0);
      }
    }

    .animate-slideUp {
      animation: slideUp 0.8s ease-out both;
    }
    ```
  </Accordion>
  <Accordion title="animate-countUp — statistic reveal">
    Used on statistic numbers. Values slide up from 30 px below while fading in over 0.8 seconds. Note: `input.css` defines this keyframe directly and overrides the `tailwind.config.js` registration.

    ```css input.css
    @keyframes countUp {
      from {
        opacity: 0;
        transform: translateY(30px);
      }
      to {
        opacity: 1;
        transform: translateY(0);
      }
    }

    .animate-countUp {
      animation: countUp 0.8s ease-out both;
    }
    ```
  </Accordion>
</AccordionGroup>

`input.css` defines additional keyframe animations beyond the three registered in `tailwind.config.js`. These are used directly via class names and do not go through Tailwind's animation utility system.

## Full tailwind.config.js

```js
/** @type {import('tailwindcss').Config} */
export default {
  content: [
    "./index.html",
    "./src/**/*.{html,js}",
  ],
  theme: {
    extend: {
      colors: {
        primary: {
          50: '#eef3f9',
          100: '#dbe6f2',
          200: '#b7cbe1',
          300: '#8ea9ca',
          400: '#5e7faa',
          500: '#1a3d73',
          600: '#163562',
          700: '#122a4d',
          800: '#0d1f39',
        },
        secondary: {
          100: '#dde8f6',
          200: '#c3d5ec',
          300: '#9eb8dc',
          400: '#6f90be',
          500: '#4d71a2',
        },
        dark: '#0a1628',
        light: '#f3f7fc',
      },
      fontFamily: {
        sans: ['Segoe UI', 'Tahoma', 'Geneva', 'Verdana', 'sans-serif'],
      },
      animation: {
        float: 'float 6s ease-in-out infinite',
        slideUp: 'slideUp 0.8s ease-out',
        countUp: 'countUp 1s ease-out',
      },
      keyframes: {
        float: {
          '0%, 100%': { transform: 'translateY(0px)' },
          '50%': { transform: 'translateY(30px)' },
        },
        slideUp: {
          from: {
            opacity: '0',
            transform: 'translateY(50px)',
          },
          to: {
            opacity: '1',
            transform: 'translateY(0)',
          },
        },
        countUp: {
          from: {
            opacity: '0',
            transform: 'scale(0.8)',
          },
          to: {
            opacity: '1',
            transform: 'scale(1)',
          },
        },
      },
    },
  },
  plugins: [],
}
```

<Tip>
  When adding a new section partial under `src/`, run `npm run dev` in a terminal before you start editing. Tailwind will watch for class changes and rebuild `output.css` automatically as you work.
</Tip>
