This guide walks you through cloning the ALESPI Solutions Web repository, installing its build tools, and running the site on your machine.

## Prerequisites

Before you begin, make sure you have:

- **Node.js 14 or later** — [nodejs.org](https://nodejs.org)
- **npm 6 or later** — included with Node.js

Verify your versions:

```bash
node --version
npm --version
```

## Setup

<Steps>
  <Step title="Clone the repository">
    Download the project source to your local machine:

    ```bash
    git clone https://github.com/Pollo184/ALESPI-Solutions-Web.git
    cd ALESPI-Solutions-Web
    ```
  </Step>
  <Step title="Install dependencies">
    Install Tailwind CSS, PostCSS, Autoprefixer, and live-server:

    ```bash
    npm install
    ```
  </Step>
  <Step title="Build CSS and start the server">
    Compile the stylesheet and launch the local development server in one command:

    ```bash
    npm run preview
    ```

    The site is now available at [http://localhost:8080](http://localhost:8080).
  </Step>
</Steps>

<Warning>
  You must serve the site through an HTTP server. Opening `index.html` directly from the filesystem (a `file://` URL) will not work because the site uses `fetch()` to load HTML partials, and browsers block cross-origin `fetch()` requests from `file://` origins.
</Warning>

## Development workflow

The project provides several npm scripts for different tasks:

| Script | Alias | What it does |
| --- | --- | --- |
| `npm run build` | `npm run build:css` | Compiles `input.css` to `output.css` once (minified) |
| `npm run dev` | `npm run watch:css` | Watches `input.css` for changes and recompiles automatically |
| `npm start` | `npm run serve` | Starts live-server at port 8080 without rebuilding CSS |
| `npm run preview` | — | Runs `build:css` then `serve` — use this for a fresh start |
| `npm test` | — | Placeholder; no automated tests are configured |

<Tip>
  When you are actively editing styles, open two terminal windows side by side. Run `npm run dev` in the first to watch for CSS changes, and `npm start` in the second to serve the site. This way the browser refreshes automatically whenever you save a change to `input.css` or any source file.
</Tip>

## What to do next

Once the site is running, you can:

- Browse to [http://localhost:8080](http://localhost:8080) and navigate the five sections: Inicio, Servicios, Expertiz, Credenciales, and Contacto.
- Edit any file under `src/` to update section content — changes appear after the live-server reloads the page.
- Modify `input.css` or `tailwind.config.js` to adjust styles — `npm run dev` will recompile automatically.
- Deploy the repository root (after running `npm run build`) to Netlify, Vercel, GitHub Pages, or any static HTTP host.
