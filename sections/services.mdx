## Card interaction

Each card has a layered hover interaction driven entirely by Tailwind utility classes and CSS transitions.

<Steps>
  <Step title="Top gradient border sweeps in">
    An absolutely-positioned `h-1` div with a `bg-gradient-to-r` from the brand palette starts at `-translate-x-full` and transitions to `translate-x-0` on hover, creating a sweeping top border effect.

    ```html servicios.html
    <div class="absolute top-0 left-0 w-full h-1
                bg-gradient-to-r from-primary-500 to-secondary-400
                -translate-x-full group-hover:translate-x-0
                transition-transform duration-500">
    </div>
    ```
  </Step>
  <Step title="Icon scales up">
    The icon container carries the `.glow-icon` class and `group-hover:scale-110 transition-transform duration-300`. A second absolutely-positioned glow overlay fades in behind the icon with `group-hover:opacity-20 blur`.
  </Step>
  <Step title="Card border activates">
    The card itself uses `border-t-4 border-transparent hover:border-primary-500` so the top border slot is always reserved but only colored on hover.
  </Step>
</Steps>

### Relevant CSS classes

| Class | Role |
|---|---|
| `.service-card` | Base card selector; used by `initializeCardObserver()` to apply scroll-triggered fade-in |
| `.card-hover` | Adds a subtle lift shadow on hover |
| `.gradient-primary` | Applies the brand gradient as a background |
| `.glow-icon` | Adds a drop-shadow glow to icon containers |

## Scroll-triggered animation

`initializeCardObserver()` in `script.js` uses the Intersection Observer API to fade cards in as you scroll them into view:

```js script.js
function initializeCardObserver() {
  const animarElementos = new IntersectionObserver((entries) => {
    entries.forEach((entry) => {
      if (entry.isIntersecting) {
        entry.target.style.opacity = '1';
        entry.target.style.transform = 'translateY(0)';
        animarElementos.unobserve(entry.target);
      }
    });
  }, {
    threshold: 0.1
  });

  document.querySelectorAll('.service-card, .valor-card').forEach((element) => {
    element.style.opacity = '0';
    element.style.transform = 'translateY(20px)';
    element.style.transition = 'opacity 0.6s ease-out, transform 0.6s ease-out';
    animarElementos.observe(element);
  });
}
```

Cards start at `opacity: 0` and `translateY(20px)`. Once 10% of the card enters the viewport, they transition to fully visible and in position. The observer then disconnects from each card after it has animated once.

<Note>
  The staggered CSS `animation-delay` values on each card (0.1s, 0.2s, 0.3s) provide an additional visual offset on initial page load when the `animate-scaleIn` class fires, separate from the scroll observer behavior.
</Note>
